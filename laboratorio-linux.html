🚀 Tutorial 1: Introducción a JS Linux

¿Qué es la Terminal Linux?

La terminal, también conocida como línea de comandos o shell, es una interfaz de texto que permite interactuar directamente con el sistema operativo. A diferencia de las interfaces gráficas, la terminal ofrece mayor control y eficiencia para tareas administrativas y de desarrollo.

📋 Comandos Básicos

Estos son los comandos esenciales para empezar:

  • pwd - Muestra el directorio actual
  • ls - Lista archivos y directorios
  • cd - Cambia de directorio
  • mkdir - Crea un nuevo directorio
  • touch - Crea un archivo vacío
  • cp - Copia archivos o directorios
  • mv - Mueve o renombra archivos
  • rm - Elimina archivos o directorios
# Ejemplo: Navegación básica
pwd                    # Muestra: /home/usuario
ls -la                 # Lista todos los archivos con detalles
cd Documentos          # Cambia al directorio Documentos
mkdir proyecto         # Crea directorio "proyecto"
cd proyecto            # Entra al directorio
touch archivo.txt      # Crea archivo vacío
echo "Hola Linux" > archivo.txt  # Escribe en el archivo
cat archivo.txt        # Muestra contenido del archivo

Consejos para Principiantes

  • Usa la tecla Tab para autocompletar nombres de archivos y comandos
  • Presiona Ctrl + C para cancelar un comando en ejecución
  • Usa Ctrl + L o escribe clear para limpiar la terminal
  • El historial de comandos se navega con las flechas ↑ y ↓

📁 Tutorial 2: Gestión de Archivos y Permisos

Sistema de Permisos Linux

Linux usa un sistema de permisos basado en tres categorías: propietario (owner), grupo (group) y otros (others). Cada categoría tiene permisos de lectura (r), escritura (w) y ejecución (x).

# Ver permisos de archivos
ls -l archivo.txt
# Resultado: -rw-r--r-- 1 usuario grupo 1024 Feb 15 10:30 archivo.txt

# Cambiar permisos (modo numérico)
chmod 755 script.sh    # rwxr-xr-x

⚙️ Tutorial 4: Gestión de Procesos y Sistema

Monitoreo de Procesos

# Ver procesos en ejecución
ps aux                    # Todos los procesos del sistema
ps aux | grep proceso     # Buscar proceso específico
top                       # Monitor interactivo (similar a Task Manager)
htop                      # Versión mejorada de top (si está instalado)

# Información del sistema
free -h                   # Memoria RAM disponible
df -h                     # Espacio en disco
uptime                    # Tiempo encendido y carga del sistema
uname -a                  # Información del kernel

Control de Procesos

🎯 Señales y Control 
# Señales comunes a procesos
kill PID                  # Envía señal TERM (terminar)
kill -9 PID               # Envía señal KILL (forzar terminación)
kill -15 PID              # Envía señal TERM (terminación limpia)
kill -STOP PID            # Pausar proceso
kill -CONT PID            # Continuar proceso pausado

# Trabajar en segundo plano
comando &                 # Ejecutar comando en background
jobs                      # Ver trabajos en background
fg %1                     # Traer trabajo 1 al foreground
bg %1                     # Enviar trabajo 1 al background
Ctrl + Z                  # Pausar proceso actual

Script de Ejemplo Completo

#!/bin/bash
# backup.sh - Script de backup automático

# Configuración
BACKUP_DIR="/backups"
SOURCE_DIR="/home/usuario/datos"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/backup_$DATE.tar.gz"

# Verificar si existe directorio de backup
if [ ! -d "$BACKUP_DIR" ]; then
    mkdir -p "$BACKUP_DIR"
    echo "Directorio de backup creado: $BACKUP_DIR"
fi

# Crear backup
echo "Creando backup de $SOURCE_DIR..."
tar -czf "$BACKUP_FILE" "$SOURCE_DIR" 2>/dev/null

# Verificar éxito
if [ $? -eq 0 ]; then
    echo "✓ Backup completado: $BACKUP_FILE"
    echo "Tamaño: $(du -h "$BACKUP_FILE" | cut -f1)"
else
    echo "✗ Error al crear backup"
    exit 1
fi

# Limpiar backups antiguos (más de 7 días)
find "$BACKUP_DIR" -name "backup_*.tar.gz" -mtime +7 -delete
echo "Backups antiguos eliminados"

🏗️ Tutorial 7: Proyecto Final Integrador

Proyecto: Sistema de Monitoreo de Servidores

Vamos a crear un sistema completo que monitoree múltiples servidores y genere reportes automáticos.

📊 Estructura del Proyecto 
# Estructura de directorios
monitoreo/
├── bin/                    # Scripts ejecutables
│   ├── monitor.sh         # Script principal
│   └── reporte.sh         # Generador de reportes
├── config/                # Archivos de configuración
│   └── servidores.conf    # Lista de servidores
├── logs/                  # Logs del sistema
├── reportes/              # Reportes generados
└── README.md              # Documentación

Script Principal de Monitoreo

#!/bin/bash
# monitor.sh - Sistema de monitoreo de servidores

CONFIG_FILE="config/servidores.conf"
LOG_FILE="logs/monitor_$(date +%Y%m%d).log"
REPORT_DIR="reportes"

# Crear directorios si no existen
mkdir -p logs reportes

# Función para verificar servidor
verificar_servidor() {
    local servidor=$1
    local puerto=$2
    
    echo "=== Verificando $servidor (puerto $puerto) ===" | tee -a "$LOG_FILE"
    
    # Verificar conectividad
    if ping -c 2 -W 1 "$servidor" > /dev/null 2>&1; then
        echo "✓ Ping exitoso" | tee -a "$LOG_FILE"
        
        # Verificar puerto
        if nc -z -w 2 "$servidor" "$puerto" > /dev/null 2>&1; then
            echo "✓ Puerto $puerto accesible" | tee -a "$LOG_FILE"
            return 0
        else
            echo "✗ Puerto $puerto inaccesible" | tee -a "$LOG_FILE"
            return 1
        fi
    else
        echo "✗ Servidor no responde al ping" | tee -a "$LOG_FILE"
        return 2
    fi
}

# Leer configuración
echo "Iniciando monitoreo: $(date)" | tee -a "$LOG_FILE"
echo "======================================" | tee -a "$LOG_FILE"

while IFS=',' read -r nombre servidor puerto; do
    # Saltar comentarios y líneas vacías
    [[ "$nombre" =~ ^#.*$ ]] && continue
    [[ -z "$nombre" ]] && continue
    
    verificar_servidor "$servidor" "$puerto"
    echo "" | tee -a "$LOG_FILE"
    
done < "$CONFIG_FILE"

echo "Monitoreo completado: $(date)" | tee -a "$LOG_FILE"
echo "Log guardado en: $LOG_FILE" | tee -a "$LOG_FILE"

# Generar reporte resumen
echo "=== RESUMEN DEL MONITOREO ===" > "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"
echo "Fecha: $(date)" >> "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"
echo "Total servidores verificados: $(grep -c "===" "$LOG_FILE")" >> "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"
echo "Servidores con problemas: $(grep -c "✗" "$LOG_FILE")" >> "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"
echo "" >> "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"
grep "✗" "$LOG_FILE" >> "$REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"

echo "Reporte generado en: $REPORT_DIR/resumen_$(date +%Y%m%d_%H%M).txt"

Script de Reportes Automáticos

#!/bin/bash
# reporte.sh - Genera reportes detallados

REPORT_DIR="reportes"
LOG_DIR="logs"
DAYS_TO_KEEP=30

echo "Generando reporte de rendimiento..."
echo "======================================"

# Reporte de uso de disco
echo "=== USO DE DISCO ===" > "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
df -h >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
echo "" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

# Reporte de memoria
echo "=== MEMORIA RAM ===" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
free -h >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
echo "" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

# Reporte de carga del sistema
echo "=== CARGA DEL SISTEMA ===" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
uptime >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
echo "" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

# Top 5 procesos por CPU
echo "=== TOP 5 PROCESOS (CPU) ===" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
ps aux --sort=-%cpu | head -6 >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
echo "" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

# Top 5 procesos por memoria
echo "=== TOP 5 PROCESOS (MEMORIA) ===" >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"
ps aux --sort=-%mem | head -6 >> "$REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

echo "Reporte generado: $REPORT_DIR/rendimiento_$(date +%Y%m%d).txt"

# Limpiar logs y reportes antiguos
echo "Limpiando archivos antiguos (más de $DAYS_TO_KEEP días)..."
find "$LOG_DIR" -name "*.log" -mtime +$DAYS_TO_KEEP -delete
find "$REPORT_DIR" -name "*.txt" -mtime +$DAYS_TO_KEEP -delete
echo "Limpieza completada"

Instalación y Uso

# 1. Clonar o crear la estructura
mkdir -p ~/monitoreo/{bin,config,logs,reportes}
cd ~/monitoreo

# 2. Crear los scripts
nano bin/monitor.sh    # Pegar el script de monitoreo
nano bin/reporte.sh    # Pegar el script de reportes
nano config/servidores.conf  # Configurar servidores

# 3. Dar permisos de ejecución
chmod +x bin/*.sh

# 4. Ejecutar manualmente
./bin/monitor.sh
./bin/reporte.sh

# 5. Programar con cron (ejecución automática)
crontab -e
# Agregar estas líneas:
# */30 * * * * /home/usuario/monitoreo/bin/monitor.sh
# 0 2 * * * /home/usuario/monitoreo/bin/reporte.sh
