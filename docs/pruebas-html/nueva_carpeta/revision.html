JavaScript

🧠 Introducción a JavaScript
JavaScript fue creado por Brendan Eich en 1995, mientras trabajaba en Netscape Communications Corporation.

Imagina que tu sitio web es como un escenario teatral: HTML sería el guion, CSS el vestuario y la escenografía, y JavaScript... el alma que le da movimiento, interacción y vida a cada escena.

¿Qué es JavaScript?

JavaScript es un lenguaje de programación que permite hacer que las páginas web reaccionen, se comuniquen con el usuario y se adapten dinámicamente. Desde botones que responden a clics hasta formularios inteligentes, JavaScript es quien hace que todo eso funcione.

¿Dónde se usa?

- 📱 Navegadores web (Chrome, Firefox, etc.)
- 🖥 Aplicaciones de escritorio (usando Electron)
- 📲 Apps móviles (con frameworks como React Native)
- 🌐 Servidores (con Node.js)

¿Por qué aprenderlo?
- Es uno de los lenguajes más populares del mundo
- Tiene una comunidad enorme y muchos recursos disponibles
- Te permite crear desde juegos simples hasta herramientas profesionales

¿Qué necesitas para empezar?
- Un editor de código (como VS Code, Sublime Text o incluso Acode en móvil)
- Un navegador moderno (Chrome, Firefox, Edge…)
- Muchas ganas de aprender.

Variables y Constantes

🧠 ¿Qué son las variables y constantes en JavaScript?

Son elementos fundamentales en cualquier lenguaje de programación. En JavaScript, las usamos para guardar y manipular datos que nuestra aplicación necesita para funcionar.

📦 Variables (let y var)

- Una variable es como una caja donde puedes guardar algo (un valor) y cambiarlo después si lo necesitas.
- En JavaScript moderno, usamos let (y menos frecuentemente var, por cuestiones de alcance y comportamiento).

✅ Características de let

- Puedes redefinir su valor.
- Tiene alcance de bloque, es decir, sólo existe dentro del bloque {} donde fue creada.

🧠 Aprendizaje clave:

Las variables creadas con let se pueden modificar. Esto es útil cuando el valor necesita cambiar durante la ejecución del programa. Ejemplo 1.

Usa const para valores fijos que no deben modificarse. Esto evita errores por reasignaciones accidentales. En el enlace de arriba, llamado ejemplo 1, lo dirige a los ejemplos prácticos del tema variables, constantes y variables y constantes globales y locales.

Las variables creadas con let o const tienen alcance de bloque. Solo viven dentro de la función {} donde fueron declaradas. Declarar una variable o constante en un ámbito local. Declarar una variable o constante en un ámbito global.



Declaración con let

let nombre = "Carlos";
nombre = "María"; // Reasignación permitida

Constantes con const

Ámbito de variables(LOCAL)

function calcularImpuesto() {
  // Variable LOCAL (solo existe dentro de esta función)
  let tasaImpuesto = 0.15;
  
  let precioProducto = 100;
  let impuesto = precioProducto * tasaImpuesto;
  console.log("Dentro de la función:");
  console.log("Impuesto calculado: $" + impuesto);
}

calcularImpuesto();

// Fuera de la función - ERROR
console.log("Fuera de la función:");
console.log("Tasa de impuesto: " + tasaImpuesto); // ReferenceError: tasaImpuesto is not defined

Explicación:
- tasaImpuesto es una variable local que solo existe dentro de calcularImpuesto().
- Al intentar usarla fuera de la función, genera un error porque no está definida en ese ámbito.

Ámbito de variables(GLOBAL)

// Variable GLOBAL (accesible en cualquier parte)
let saldo = 1000;

function gastarDinero() {
  // Variable LOCAL
  let gasto = 200;
  saldo -= gasto; // Accede a la variable global
  console.log("Gastaste $" + gasto);
}

console.log("Saldo inicial: $" + saldo); // 1000
gastarDinero();
console.log("Saldo actualizado: $" + saldo); //800

// Intento acceder a variable local desde fuera - ERROR
console.log("Último gasto: $" + gasto); // ReferenceError: gasto is not defined

Explicación:
1. saldo es global:
- Accesible dentro y fuera de funciones.
- Modificarla en gastarDinero() afecta su valor globalmente.
2. gasto es local:
- Solo existe dentro de gastarDinero().
- Inaccesible fuera de la función.

💡 Buenas prácticas para nombrar variables y organizar el código.

✅ Usa nombres claros y descriptivos

Como por ejemplo:
let temperaturaActual = 22; // ✔️ Mejor que "temp" o "x"

✅ Escribe en camelCase para variables y constantes

Como por ejemplo:
const valorEdad = 60;
let nombreUsuario = "Eduardo";

console.log("Hola yo me llamo " + nombreUsuario + " y tengo" + valorEdad + " de edad" );

✅ Evita abreviaciones innecesarias

Por ejemplo:
let cnt = 5; // ❌ ¿Qué significa "cnt"?
let cantidad = 5; // ✔️ Claro y entendible

✅ Agrupa variables relacionadas. Esto facilita la lectura del código y su mantenimiento.

Por ejemplo:
let nombre = document.getElementById("nombre-input").value.trim();
let edad = document.getElementById("edad-input").value.trim();
let ciudad = document.getElementById("ciudad-input").value.trim();

Tipo de variables en JavaScript

En JavaScript, cada dato que guardamos tiene un tipo, y conocerlos te permite usar las funciones correctas, hacer comparaciones y controlar el flujo de tu programa.

📌 Tipos primitivos

-Tipo | Ejemplo | Descripción

String | "Hola mundo" | Texto entre comillas.

-Number | 42, 3.14 | Números enteros o decimales.

-Boolean | true, false | Verdadero o falso (decisiones).

-Null | null | Sin valor (intencionadamente vacío).

-Undefined | undefined | Valor no asignado.

-BigInt | 9007199254740991n | Números extremadamente grandes.

Tipo | Ejemplo | Descripción

💡 Objetivo: Que el estudiante use typeof para explorar y clasificar los tipos de datos en Java Script.

Typeof es una palabra clave reservada en JavaScript. Su propósito principal es ayudarte a identificar el tipo de dato de una variable o expresión en tiempo de ejecución.

🔍 ¿Cómo funciona typeof?

La sintaxis básica es:
`javascript
typeof valor;

Por ejemplo:
typeof "Hola"; // "string"
typeof 42; // "number"
typeof true; // "boolean"
typeof undefined; // "undefined"
typeof null; // "object" ← curioso, ¿no?
typeof {}; // "object"
typeof []; // "object"
typeof function(){}; // "function"

` 🧠 Detalles interesantes:

- Aunque null es técnicamente un tipo primitivo, typeof null devuelve "object" por una razón histórica en el lenguaje.
- Para arrays y otros objetos más específicos, typeof no es suficiente por sí solo. Se usa junto con otras técnicas como Array.isArray() o instanceof.

💡 Y sí, como palabra reservada, no puedes usar typeof como nombre de variable o función:

Por ejemplo en JavaScript
let typeof = 123; // ❌ Error

🧮 Operadores en JavaScript.

Los operadores son símbolos que permiten realizar operaciones con valores o variables. Se dividen en varias categorías:

---

➕ Operadores Aritméticos .

Operador | Símbolo | Ejemplo | Resultado

-Suma | + | 3 + 2 | 5
-Resta | - | 5 - 2 | 3
-Multip.| × | 4 × 2 | 8

let resultado9 = 2 + 3  2;    // Primero exponente: 2 + 9 = 11

// =============================================
// EJEMPLOS DE OPERADORES ARITMÉTICOS EN JAVASCRIPT
// =============================================

// 1. SUMA (+) - Concatenación o adición numérica
// Problema: Ana tiene 5 manzanas y compra 3 más. ¿Cuántas tiene en total?
let manzanasIniciales = 5;
let manzanasCompradas = 3;
let totalManzanas = manzanasIniciales + manzanasCompradas;
console.log(Total manzanas: ${totalManzanas}); // Output: 8

// 2. RESTA (-) - Diferencia entre valores
// Problema: Carlos tenía $50, gastó $35 en libros. ¿Cuánto le queda?
let dineroInicial = 50;
let gastoLibros = 35;
let dineroRestante = dineroInicial - gastoLibros;
console.log(Dinero restante: $${dineroRestante}); // Output: 15

// 3. MULTIPLICACIÓN (*) - Producto de valores
// Problema: Cada caja tiene 24 latas de refresco. ¿Cuántas latas hay en 5 cajas?
let latasPorCaja = 24;
let numeroCajas = 5;
let totalLatas = latasPorCaja * numeroCajas;
console.log(Total latas: ${totalLatas}); // Output: 120

// 4. DIVISIÓN (/) - Cociente entre valores
// Problema: Se reparten 150 caramelos entre 25 niños. ¿Cuántos tocan por niño?
let totalCaramelos = 150;
let numeroNiños = 25;
let caramelosPorNiño = totalCaramelos / numeroNiños;
console.log(Caramelos por niño: ${caramelosPorNiño}); // Output: 6

// 5. MÓDULO (%) - Resto de una división
// Problema: Se empaquetan 47 huevos en cartones de 12. ¿Cuántos sobran?
let totalHuevos = 47;
let capacidadCarton = 12;
let huevosSobrantes = totalHuevos % capacidadCarton;
console.log(Huevos sobrantes: ${huevosSobrantes}); // Output: 11

// 6. EXPONENCIACIÓN () - Potencia de un número
// Problema: Un cubo mide 3 cm por lado. ¿Cuál es su volumen?
let arista = 3;
let volumenCubo = arista  3; // 3 elevado al cubo
console.log(Volumen del cubo: ${volumenCubo} cm³); // Output: 27

// 7. INCREMENTO (++) - Aumentar en 1 unidad
// Problema: Contador de visitas a una página web
let visitas = 100;
console.log(Visitas iniciales: ${visitas});
visitas++; // Incremento POST: usa el valor actual luego suma
console.log(Después de una visita: ${visitas}); // Output: 101

// Pre-incremento: suma primero luego usa el valor
let contador = 10;
let nuevoValor = ++contador;
console.log(Valor con pre-incremento: ${nuevoValor}); // Output: 11

// 8. DECREMENTO (--) - Reducir en 1 unidad
// Problema: Inventario de productos al vender una unidad
let stock = 50;
console.log(Stock inicial: ${stock});
stock--; // Decremento POST: usa el valor actual luego resta
console.log(Stock después de venta: ${stock}); // Output: 49

// Pre-decremento: resta primero luego usa el valor
let temperatura = 30;
let temperaturaAjustada = --temperatura;
console.log(Temperatura ajustada: ${temperaturaAjustada}); // Output: 29

🔀 Condicionales: Tomando decisiones

Las estructuras condicionales permiten que tu programa tome decisiones basadas en condiciones. Son como las bifurcaciones en un camino: dependiendo de ciertas condiciones, el programa seguirá un camino u otro.

Buenas prácticas
✅ Usa condiciones claras y descriptivas
✅ Anida condicionales solo cuando sea necesario
✅ Considera usar switch cuando tengas múltiples casos
✅ Evita condiciones demasiado complejas (más de 3 operadores)

if, else if, else

// Verificar la hora del día
let hora = new Date().getHours();
let mensaje;

if (hora < 12) {
    mensaje = "¡Buenos días!";
} else if (hora < 19) {
    mensaje = "¡Buenas tardes!";
} else {
    mensaje = "¡Buenas noches!";
}

console.log(mensaje);

// Forma compacta de if/else
let edad = 20;

Switch

// Determinar el día de la semana
let dia = new Date().getDay();

    case 0: nombreDia = "Domingo"; break;

    case 2: nombreDia = "Martes"; break;

    case 4: nombreDia = "Jueves"; break;

    default: nombreDia = "Día inválido";
}

console.log(Hoy es ${nombreDia});

function calcularFactorial(n) {
    if (n < 0) return NaN; // NaN indica "Not a Number" (matemáticamente inválido)
    let resultado = 1;
    for (let i = 1; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
}

function calcularFactorial(n) {
    if (n < 0) return NaN; // NaN = "Not a Number" (estándar matemático)
    if (n === 0 || n === 1) return 1; // Casos base optimizados
    
    let resultado = 1;
    let i = 2; // Comenzar desde 2 (ahorra una iteración)
    while (i <= n) {
        resultado *= i;
        i++;
    }
    return resultado;
}
console.log(calcularFactorial(5));  // 120
console.log(calcularFactorial(0));  // 1
console.log(calcularFactorial(1));  // 1
console.log(calcularFactorial(-1)); // NaN (correcto)

✅ Permiten organizar el código en tareas más pequeñas y manejables
✅ Evitan la repetición de código (principio DRY - Don't Repeat Yourself)
✅ Facilitan la depuración y mantenimiento del código
✅ Son ciudadanos de primera clase (pueden asignarse, pasarse y retornarse)

En JavaScript, las funciones pueden definirse de varias formas y tienen características únicas que las hacen extremadamente poderosas.

🧩 ¿Qué es un método en JavaScript?

Un método es simplemente una función que está asociada a un objeto. Es como una habilidad que ese objeto puede ejecutar. Por ejemplo si vemos a un chofer como objeto JavaScript, su habilidad para conducir, sería un método. Si vemos un aerostático como objeto JavaScript, su capacidad de suspenderse en el aire y volar seria un método.

🔍 Diferencia entre función y método:
- Una función es independiente: puede existir por sí sola.
- Un método vive dentro de un objeto y se invoca a través de ese objeto.

Que es toFixed()? Empleado en el ejemplo anterior.

toFixed() es un método en JavaScript que se utiliza para formatear un número y devolverlo como una cadena (string), con un número específico de dígitos después del punto decimal.

Sintaxis:
`javascript
numero.toFixed(cantidadDecimales);
`
Características:
1. Redondeo: Si el número de decimales especificado es menor que los decimales reales del número, redondea el valor.
2. Relleno con ceros: Si los decimales reales son menores que los especificados, completa con ceros.
3. Retorna un string: El resultado es una cadena, no un número.

Ejemplos:
`javascript
let num = 3.1416;
console.log(num.toFixed(2)); // "3.14"
console.log(num.toFixed(3)); // "3.142" (redondeo hacia arriba)
console.log(num.toFixed(6)); // "3.141600" (rellena con ceros)
`
⚠️ Importante:
- Si el número es mayor que 1e21, toFixed() devuelve el número en notación científica.
- Si no se especifican decimales, usa 0 (ej: num.toFixed() → "3").

// Tagged template: función que procesa un template literal
function etiqueta(strings, ...variables) {
  return strings.reduce((resultado, str, i) => {
    return resultado + str + (variables[i] ? [${variables[i]}] : '');
  }, '');
}

const producto = "teclado";
const precio = 45;

// Uso del tagged template
const mensaje = etiquetaEl ${producto} cuesta ${precio}€.;
console.log(mensaje); // "El [teclado] cuesta [45]€."

Explicación:

Los backticks habilitan tagged templates, donde una función personalizada procesa el string y sus variables.

🏷️ ¿Qué es un tagged template?

Es cuando una función precede a una plantilla literal. En lugar de que JavaScript construya la cadena directamente, llama a esa función con los fragmentos de texto y las expresiones interpoladas por separado.

Ejemplo de tagged templates


function resaltar(strings, ...valores) {
  return strings.reduce((resultado, parte, i) => {
    const valor = valores[i] !== undefined ? ${valores[i]} : "";
    return resultado + parte + valor;
  }, "");
}

const nombre = "Eduardo";
const lenguaje = "JavaScript";

const mensaje = resaltarHola ${nombre}, estás aprendiendo ${lenguaje}.;
console.log(mensaje);

Vamos a comentar acerca de, .reduce() en JavaScript. Es un método de arrays que se utiliza para reducir todos los elementos de un array a un único valor, aplicando una función acumuladora de manera iterativa. Es especialmente útil para operaciones como sumar elementos, combinar datos, o transformar estructuras.

Sintaxis básica:
`javascript
array.reduce((acumulador, elementoActual, índice, array) => { // Lógica para combinar el acumulador y el elemento actual }, valorInicial);
`
- acumulador: Valor que se va acumulando en cada iteración.
- elementoActual: El elemento del array que se está procesando.
- índice (opcional): La posición actual en el array.
- array (opcional): El array original.
- valorInicial (opcional): Valor inicial del acumulador. Si no se especifica, se usa el primer elemento del array.
---
Ejemplos comunes:
1. Sumar todos los elementos de un array:
const números = [1, 2, 3, 4]; const suma = números.reduce((acum, num) => acum + num, 0); console.log(suma); // 10 `

2. Concatenar strings:
`javascript
const palabras = ["Hola", "mundo", "!"]; const frase = palabras.reduce((acum, palabra) => acum + " " + palabra, ""); console.log(frase.trim()); // "Hola mundo !" `

3. Contar ocurrencias de elementos:
`javascript
const frutas = ["manzana", "banana", "manzana", "naranja"]; const conteo = frutas.reduce((acum, fruta) => { acum[fruta] = (acum[fruta] || 0) + 1; return acum; }, {});
console.log(conteo); // { manzana: 2, banana: 1, naranja: 1 }
`
4. Aplanar un array de arrays:
`javascript
const arrays = [[1, 2], [3, 4], [5]]; const plano = arrays.reduce((acum, arr) => acum.concat(arr), []); console.log(plano); // [1, 2, 3, 4, 5]
`
---
¡Importante!
- Si no se proporciona un valorInicial, el acumulador tomará el primer valor del array y la iteración comenzará desde el segundo elemento.
- Si el array está vacío y no hay valorInicial, .reduce() lanzará un error.
`javascript
const vacío = []; vacío.reduce((acum, num) => acum + num); // ¡Error! `
---
Resumen:
- Propósito: Transformar un array en un único valor.
- Flexibilidad: Puede manejar desde operaciones matemáticas hasta transformaciones complejas de datos.
- Legibilidad: Suele ser más declarativo que usar bucles for o forEach.

Que es .trim() en JavaScript: es un método que se utiliza para eliminar los espacios en blanco (o caracteres de whitespace) al inicio y al final de una cadena de texto (string). Los espacios en blanco incluyen espacios, tabs, saltos de línea, etc.

Ejemplo básico:
`javascript
let texto = " Hola mundo! "; let textoTrimmed = texto.trim(); console.log(textoTrimmed); // Output: "Hola mundo!"
`
Características importantes:
- No modifica la cadena original (los strings en JavaScript son inmutables).
- Solo elimina espacios al inicio y al final, no los espacios entre palabras.

También existen variantes:
- .trimStart() o .trimLeft(): Elimina espacios solo al inicio.
- .trimEnd() o .trimRight(): Elimina espacios solo al final.

Ejemplo con trimStart() y trimEnd():
`javascript
let texto = " Hola "; console.log(texto.trimStart()); // "Hola " console.log(texto.trimEnd()); // " Hola"
`
💡 ¿Para qué se usa?
Es muy útil cuando recibes datos de usuarios (como inputs en formularios) y quieres asegurarte de que no haya espacios accidentalmente al inicio o final.

La herencia prototípica en JavaScript es un mecanismo mediante el cual un objeto puede heredar propiedades y métodos de otro objeto, conocido como su prototipo. A diferencia de la herencia basada en clases (como en Java o C++), JavaScript utiliza prototipos para delegar acceso a propiedades y métodos.

¿Cómo funciona?
Cada objeto en JavaScript tiene una propiedad interna [[Prototype]]. Cuando intentas acceder a una propiedad o método en un objeto, JavaScript busca primero en el objeto mismo. Si no la encuentra, busca en su prototipo, luego en el prototipo del prototipo, y así sucesivamente hasta llegar al objeto null (fin de la cadena).

Crear herencia prototípica: Existen varias formas de establecer una cadena de prototipos:

1. Usando Object.create() (Recomendado): Crea un nuevo objeto cuyo prototipo es el objeto especificado.

Ejemplo de herencia de prototipos


  const animal = {
  sonido: function() {
    console.log("Hace un sonido");
  }
};

// Crear un objeto perro que hereda de animal
const perro = Object.create(animal);
perro.ladrar = function() {
  console.log("Guau guau");
};

perro.sonido(); // "Hace un sonido" (heredado)
perro.ladrar(); // "Guau guau" (propio)

- Las funciones regulares tienen su propio this que referencia al objeto que contiene el método
- Las arrow functions no tienen su propio this, heredan el this del contexto donde fueron definidas

Promesas y async/await

Para manejar operaciones asíncronas. Las promesas representan un valor futuro, y async/await permite escribir código asíncrono de forma síncrona.

Promesa Básica

const miPromesa = new Promise((resolve, reject) => {
    setTimeout(() => {
        const exito = true;
        exito ? resolve("¡Operación exitosa!") : reject("Error");
    }, 2000);
});

miPromesa
    .then(resultado => console.log(resultado))
    .catch(error => console.error(error));

Async/Await

function esperar(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
}

async function ejecutar() {
    console.log("Inicio");
    await esperar(2000);
    console.log("Después de 2 segundos");
}

ejecutar();

Manejo de Errores

try/catch permite manejar excepciones de forma controlada. Puedes lanzar errores personalizados con throw y usar finally para código que siempre debe ejecutarse.

Try/Catch Básico

try {
    // Código que puede fallar
    let resultado = 10 / 0;
    if (!isFinite(resultado)) {
        throw new Error("División por cero");
    }
} catch (error) {
    console.error("Ocurrió un error:", error.message);
} finally {
    console.log("Esto siempre se ejecuta");
}

Error Personalizado

class MiError extends Error {
    constructor(mensaje) {
        super(mensaje);
        this.name = "MiError";
    }
}

function validarEdad(edad) {
    if (edad < 18) throw new MiError("Menor de edad");
    return true;
}

try {
    validarEdad(15);
} catch (e) {
    if (e instanceof MiError) {
        console.error("Error personalizado:", e.message);
    }
}

Promise con Catch

function operacionRiesgosa() {
    return new Promise((resolve, reject) => {
        const exito = Math.random() > 0.5;
        exito ? resolve("¡Éxito!") : reject("Fallo aleatorio");
    });
}

operacionRiesgosa()
    .then(console.log)
    .catch(error => console.error("Error:", error));
