JavaScript

🧠 Introducción a JavaScript

JavaScript fue creado por Brendan Eich en 1995, mientras trabajaba en Netscape Communications Corporation.

Imagina que tu sitio web es como un escenario teatral: HTML sería el guion, CSS el vestuario y la escenografía, y JavaScript... el alma que le da movimiento, interacción y vida a cada escena.

¿Qué es JavaScript?

JavaScript es un lenguaje de programación que permite hacer que las páginas web reaccionen, se comuniquen con el usuario y se adapten dinámicamente. Desde botones que responden a clics hasta formularios inteligentes, JavaScript es quien hace que todo eso funcione.

¿Dónde se usa?

- 📱 Navegadores web (Chrome, Firefox, etc.)
- 🖥 Aplicaciones de escritorio (usando Electron)
- 📲 Apps móviles (con frameworks como React Native)
- 🌐 Servidores (con Node.js)

¿Por qué aprenderlo?
- Es uno de los lenguajes más populares del mundo
- Tiene una comunidad enorme y muchos recursos disponibles
- Te permite crear desde juegos simples hasta herramientas profesionales

¿Qué necesitas para empezar?
- Un editor de código (como VS Code, Sublime Text o incluso Acode en móvil)
- Un navegador moderno (Chrome, Firefox, Edge…)
- Muchas ganas de aprender.

Variables y Constantes

🧠 ¿Qué son las variables y constantes en JavaScript?

Son elementos fundamentales en cualquier lenguaje de programación. En JavaScript, las usamos para guardar y manipular datos que nuestra aplicación necesita para funcionar.

📦 Variables (let y var)

- Una variable es como una caja donde puedes guardar algo (un valor) y cambiarlo después si lo necesitas.
- En JavaScript moderno, usamos let (y menos frecuentemente var, por cuestiones de alcance y comportamiento).

✅ Características de let

- Puedes redefinir su valor.
- Tiene alcance de bloque, es decir, sólo existe dentro del bloque {} donde fue creada.

🧠 Aprendizaje clave:

Las variables creadas con let se pueden modificar. Esto es útil cuando el valor necesita cambiar durante la ejecución del programa. Ejemplo 1.

Usa const para valores fijos que no deben modificarse. Esto evita errores por reasignaciones accidentales. En el enlace de arriba, llamado ejemplo 1, lo dirige a los ejemplos prácticos del tema variables, constantes y variables y constantes globales y locales.

Las variables creadas con let o const tienen alcance de bloque. Solo viven dentro de la función {} donde fueron declaradas. Declarar una variable o constante en un ámbito local. Declarar una variable o constante en un ámbito global.



Declaración con let

let nombre = "Carlos";
nombre = "María"; // Reasignación permitida
console.log("Hola " + nombre);

Constantes con const

const PI = 3.1416;
// PI = 3.14; // Error!
console.log("Valor de PI: " + PI);

Ámbito de variables(LOCAL)

function calcularImpuesto() {
  // Variable LOCAL (solo existe dentro de esta función)
  let tasaImpuesto = 0.15;
  
  let precioProducto = 100;
  let impuesto = precioProducto * tasaImpuesto;
  console.log("Dentro de la función:");
  console.log("Impuesto calculado: $" + impuesto);
}

calcularImpuesto();

// Fuera de la función - ERROR
console.log("Fuera de la función:");
console.log("Tasa de impuesto: " + tasaImpuesto); // ReferenceError: tasaImpuesto is not defined

Explicación:
- `tasaImpuesto` es una variable **local** que solo existe dentro de `calcularImpuesto()`.
- Al intentar usarla fuera de la función, genera un error porque no está definida en ese ámbito.

Ámbito de variables(GLOBAL)

// Variable GLOBAL (accesible en cualquier parte)
let saldo = 1000;

function gastarDinero() {
  // Variable LOCAL
  let gasto = 200;
  saldo -= gasto; // Accede a la variable global
  console.log("Gastaste $" + gasto);
}

console.log("Saldo inicial: $" + saldo); // 1000
gastarDinero();
console.log("Saldo actualizado: $" + saldo); //800

// Intento acceder a variable local desde fuera - ERROR
console.log("Último gasto: $" + gasto); // ReferenceError: gasto is not defined

Explicación:
1. `saldo` es **global**:
- Accesible dentro y fuera de funciones.
- Modificarla en `gastarDinero()` afecta su valor globalmente.
2. `gasto` es **local**:
- Solo existe dentro de `gastarDinero()`.
- Inaccesible fuera de la función.

💡 Buenas prácticas para nombrar variables y organizar el código.

✅ Usa nombres claros y descriptivos

Como por ejemplo:
let temperaturaActual = 22; // ✔️ Mejor que "temp" o "x"

✅ Escribe en camelCase para variables y constantes

Como por ejemplo:
const valorEdad = 60;
let nombreUsuario = "Eduardo";

console.log("Hola yo me llamo " + nombreUsuario + " y tengo" + valorEdad + " de edad" );

✅ Evita abreviaciones innecesarias

Por ejemplo:
let cnt = 5; // ❌ ¿Qué significa "cnt"?
let cantidad = 5; // ✔️ Claro y entendible

✅ Agrupa variables relacionadas. Esto facilita la lectura del código y su mantenimiento.

Por ejemplo:
let nombre = document.getElementById("nombre-input").value.trim();
let edad = document.getElementById("edad-input").value.trim();
let ciudad = document.getElementById("ciudad-input").value.trim();

🎯 Actividad 1: Diferenciar tipos de datos.

💡 Objetivo: Que el estudiante use typeof para explorar y clasificar los tipos de datos en Java Script.

Typeof es una palabra clave reservada en JavaScript. Su propósito principal es ayudarte a identificar el tipo de dato de una variable o expresión en tiempo de ejecución.

🔍 ¿Cómo funciona typeof?

La sintaxis básica es:
`javascript
typeof valor;

Por ejemplo:
typeof "Hola"; // "string"
typeof 42; // "number"
typeof true; // "boolean"
typeof undefined; // "undefined"
typeof null; // "object" ← curioso, ¿no?
typeof {}; // "object"
typeof []; // "object"
typeof function(){}; // "function"

` 🧠 Detalles interesantes:

- Aunque null es técnicamente un tipo primitivo, typeof null devuelve "object" por una razón histórica en el lenguaje.
- Para arrays y otros objetos más específicos, typeof no es suficiente por sí solo. Se usa junto con otras técnicas como Array.isArray() o instanceof.

💡 Y sí, como palabra reservada, no puedes usar typeof como nombre de variable o función:

Por ejemplo en JavaScript
let typeof = 123; // ❌ Error

console.log(resultado3);  // Output: 1

// 4. Exponentes negativos (fracciones)
let resultado4 = 3 ** -2;  // Equivale a 1/(3^2) = 1/9
console.log(resultado4);  // Output: 0.111111...

let resultado5 = 16 ** (1/2);  // Raíz cuadrada de 16
console.log(resultado5);  // Output: 4

let resultado6 = 8 ** (1/3);  // Raíz cúbica de 8
console.log(resultado6);  // Output: 2

// 6. Bases decimales
let resultado7 = 2.5 ** 2;  // 2.5 elevado al cuadrado (2.5*2.5)
console.log(resultado7);  // Output: 6.25

// 7. Precedencia de operadores (uso de paréntesis)
let resultado8 = (2 + 3) ** 2;  // Primero suma: 5^2 = 25
let resultado9 = 2 + 3 ** 2;    // Primero exponente: 2 + 9 = 11
console.log(resultado8, resultado9);  // Output: 25 11

// 8. Asignación compuesta (**=)
let numero = 3;
numero **= 2;  // Equivale a: numero = numero ** 2
console.log(numero);  // Output: 9

🔀 Condicionales: Tomando decisiones

Las estructuras condicionales permiten que tu programa tome decisiones basadas en condiciones. Son como las bifurcaciones en un camino: dependiendo de ciertas condiciones, el programa seguirá un camino u otro.

Buenas prácticas
✅ Usa condiciones claras y descriptivas
✅ Anida condicionales solo cuando sea necesario
✅ Considera usar switch cuando tengas múltiples casos
✅ Evita condiciones demasiado complejas (más de 3 operadores)

if, else if, else

// Verificar la hora del día
let hora = new Date().getHours();
let mensaje;

if (hora < 12) {
    mensaje = "¡Buenos días!";
} else if (hora < 19) {
    mensaje = "¡Buenas tardes!";
} else {
    mensaje = "¡Buenas noches!";
}

console.log(mensaje);

Operador Ternario

// Forma compacta de if/else
let edad = 20;
let puedeConducir = (edad >= 18) ? "Sí, puede conducir" : "No puede conducir";

console.log(puedeConducir);

Switch

// Determinar el día de la semana
let dia = new Date().getDay();

switch(dia) {
    case 0: nombreDia = "Domingo"; break;
    case 1: nombreDia = "Lunes"; break;
    case 2: nombreDia = "Martes"; break;
    case 3: nombreDia = "Miércoles"; break;
    case 4: nombreDia = "Jueves"; break;
    case 5: nombreDia = "Viernes"; break;
    case 6: nombreDia = "Sábado"; break;
    default: nombreDia = "Día inválido";
}

console.log(`Hoy es ${nombreDia}`);

Funcionamiento:
- Inicialización: `let i = 0` (se ejecuta una vez al inicio)
- Condición: `i < 5` (verificada antes de cada iteración)
- Expresión final: `i++` (ejecutada después de cada iteración)

Usos: Ideal cuando sabes cuántas iteraciones necesitas.

2. Bucle `while`

Funcionamiento:
- Verifica la condición antes de ejecutar el bloque.
- Si la condición es falsa inicialmente, no se ejecuta.

Usos: Cuando el número de iteraciones es incierto (ej.: leer datos hasta encontrar `null`).

3. Bucle `do...while` Funcionamiento:
- Ejecuta el bloque al menos una vez.
- Verifica la condición después de ejecutar.

Usos: Menús interactivos donde la acción debe ejecutarse antes de validar.

4. Bucle `for...of` Funcionamiento:
- Itera sobre valores de objetos iterables (arrays, strings, mapas, etc.).

- No funciona con objetos genéricos.

Usos: Procesar elementos de arrays/colecciones sin necesidad de índices.

5. Bucle `for...in`

Funcionamiento:
- Itera sobre claves (keys) de un objeto.
- Incluye propiedades heredadas (usar `hasOwnProperty()` para filtrar).

Usos: Inspeccionar propiedades de objetos (no recomendado para arrays).

6. Método `forEach()` (para arrays)

Funcionamiento:
- Método de arrays que ejecuta una función por cada elemento.
- No se puede romper con `break` (usa `for...of` si necesitas salir temprano).

Usos: Transformar datos en arrays (funcional).

Problema 1: Calculadora de Factorial

Descripción:
Crear una función que calcule el factorial de un número. El factorial de `n` (n!) es el producto de todos los enteros positivos desde 1 hasta `n`.

Primera solución de ejercicio de cálculo de factorial con for de incremento. 

function calcularFactorial(n) {
    if (n < 0) return NaN; // NaN indica "Not a Number" (matemáticamente inválido)
    let resultado = 1;
    for (let i = 1; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
}
console.log(calcularFactorial(5));  // 120
console.log(calcularFactorial(0));  // 1
console.log(calcularFactorial(-1)); // NaN (correcto, no definido)

Segunda solución para ejercicio de cálculo de factorial con while.

function calcularFactorial(n) {
    if (n < 0) return NaN; // NaN = "Not a Number" (estándar matemático)
    if (n === 0 || n === 1) return 1; // Casos base optimizados
    
    let resultado = 1;
    let i = 2; // Comenzar desde 2 (ahorra una iteración)
    while (i <= n) {
        resultado *= i;
        i++;
    }
    return resultado;
}
console.log(calcularFactorial(5));  // 120
console.log(calcularFactorial(0));  // 1
console.log(calcularFactorial(1));  // 1
console.log(calcularFactorial(-1)); // NaN (correcto)

function calcularFactorial(n) {
    if (n < 0) return NaN; // NaN = "Not a Number"

    let resultado = 1;
    for (let i = n; i >= 1; i--) {
        resultado *= i;
    }
    return resultado;
}

Vamos a comentar acerca de, **`.reduce()`** en JavaScript. Es un método de arrays que se utiliza para **reducir** todos los elementos de un array a un **único valor**, aplicando una función acumuladora de manera iterativa. Es especialmente útil para operaciones como sumar elementos, combinar datos, o transformar estructuras.

Sintaxis básica:
```javascript
array.reduce((acumulador, elementoActual, índice, array) => { // Lógica para combinar el acumulador y el elemento actual }, valorInicial);
```
- **`acumulador`**: Valor que se va acumulando en cada iteración.
- **`elementoActual`**: El elemento del array que se está procesando.
- **`índice`** (opcional): La posición actual en el array.
- **`array`** (opcional): El array original.
- **`valorInicial`** (opcional): Valor inicial del acumulador. Si no se especifica, se usa el primer elemento del array.
---
Ejemplos comunes:
1. **Sumar todos los elementos de un array**:
const números = [1, 2, 3, 4]; const suma = números.reduce((acum, num) => acum + num, 0); console.log(suma); // 10 ```

2. **Concatenar strings**:
```javascript
const palabras = ["Hola", "mundo", "!"]; const frase = palabras.reduce((acum, palabra) => acum + " " + palabra, ""); console.log(frase.trim()); // "Hola mundo !" ```

3. **Contar ocurrencias de elementos**:
```javascript
const frutas = ["manzana", "banana", "manzana", "naranja"]; const conteo = frutas.reduce((acum, fruta) => { acum[fruta] = (acum[fruta] || 0) + 1; return acum; }, {});
console.log(conteo); // { manzana: 2, banana: 1, naranja: 1 }
```
4. **Aplanar un array de arrays**:
```javascript
const arrays = [[1, 2], [3, 4], [5]]; const plano = arrays.reduce((acum, arr) => acum.concat(arr), []); console.log(plano); // [1, 2, 3, 4, 5]
```
---
¡Importante!
- Si **no se proporciona** un `valorInicial`, el acumulador tomará el primer valor del array y la iteración comenzará desde el segundo elemento.
- Si el array está **vacío** y no hay `valorInicial`, `.reduce()` lanzará un error.
```javascript
const vacío = []; vacío.reduce((acum, num) => acum + num); // ¡Error! ```
---
Resumen:
- **Propósito**: Transformar un array en un único valor.
- **Flexibilidad**: Puede manejar desde operaciones matemáticas hasta transformaciones complejas de datos.
- **Legibilidad**: Suele ser más declarativo que usar bucles `for` o `forEach`.

La **herencia prototípica** en JavaScript es un mecanismo mediante el cual un objeto puede heredar propiedades y métodos de otro objeto, conocido como su **prototipo**. A diferencia de la herencia basada en clases (como en Java o C++), JavaScript utiliza prototipos para delegar acceso a propiedades y métodos.

¿Cómo funciona?
Cada objeto en JavaScript tiene una propiedad interna `[[Prototype]]`. Cuando intentas acceder a una propiedad o método en un objeto, JavaScript busca primero en el objeto mismo. Si no la encuentra, busca en su prototipo, luego en el prototipo del prototipo, y así sucesivamente hasta llegar al objeto `null` (fin de la cadena).

1. Diferentes sintaxis para definir métodos:

- **Métodos tradicionales**: `metodo: function() { ... }`
- **Shorthand de ES6**: `metodo() { ... }` (más conciso y legible)

3. Diferencias importantes:

- Las funciones regulares tienen su propio `this` que referencia al objeto que contiene el método
- Las arrow functions no tienen su propio `this`, heredan el `this` del contexto donde fueron definidas

Parte 1: Palabras Reservadas en JavaScript

Las **palabras reservadas** (o keywords) son términos que tienen un significado especial para el lenguaje JavaScript. No puedes usarlas como nombres de variables, funciones, etiquetas o identificadores.

Se dividen en varias categorías:

A. Palabras Reservadas Actuales (Usadas) Estas palabras forman parte activa de la sintaxis del lenguaje.

* **`break`**: Termina un bucle (`for`, `while`, `do...while`) o una sentencia `switch`.
* **`case`**: Define una condición dentro de una sentencia `switch`.
* **`catch`**: Se utiliza en bloques `try...catch` para manejar excepciones (errores) que ocurran en el bloque `try`.
* **`class`**: Declara una clase, una plantilla para crear objetos.
* **`const`**: Declara una constante (una variable cuyo valor no puede ser reasignado).
* **`continue`**: Omite el resto de la iteración actual de un bucle y continúa con la siguiente.
* **`debugger`**: Invoca cualquier funcionalidad de depuración disponible. Si no hay depurador, no tiene efecto.
* **`default`**: 1) Especifica el código a ejecutar en un `switch` si ningún `case` coincide. 2) Se usa en `export` para exportar un valor por defecto de un módulo.
* **`delete`**: Elimina una propiedad de un objeto o un elemento de un array.
* **`do`**: Parte del bucle `do...while`, que ejecuta un bloque de código al menos una vez antes de comprobar la condición.
* **`else`**: Ejecuta un bloque de código si la condición de la sentencia `if` es falsa.
* **`export`**: Se utiliza en módulos para exportar funciones, objetos o valores primitivos para que puedan ser usados por otros scripts con `import`.
* **`extends`**: Se usa en una declaración de clase para crear una clase hija (herencia).
* **`finally`**: Se utiliza en bloques `try...catch` para ejecutar código después del bloque, regardlesse de si hubo un error o no.
* **`for`**: Crea un bucle que consiste en tres expresiones opcionales dentro de paréntesis, seguidas de un bloque de código a ejecutar.
* **`function`**: Declara una función.
* **`if`**: Ejecuta un bloque de código si una condición especificada es verdadera.
* **`import`**: Se utiliza en módulos para importar bindings exportados desde otro módulo.
* **`in`**: Devuelve `true` si la propiedad especificada está en el objeto o en su cadena de prototipos.
* **`instanceof`**: Devuelve `true` si el objeto especificado es de el tipo de objeto especificado.
* **`new`**: Crea una instancia de un objeto definido por el usuario o de un tipo predefinido.
* **`return`**: Termina la ejecución de una función y especifica un valor para ser devuelto al llamador de esa función.
* **`super`**: Se usa para llamar a funciones del padre de un objeto, often en constructores de clases.
* **`switch`**: Evalúa una expresión y ejecuta el código asociado a el `case` que coincida.
* **`this`**: Se refiere al objeto desde el cual se llamó a una función o método. Su valor depende del contexto.
* **`throw`**: Lanza una excepción (error) definida por el usuario.
* **`try`**: Define un bloque de código para intentar ejecutar y del cual se pueden manejar posibles errores.
* **`typeof`**: Devuelve una cadena que indica el tipo del operando sin evaluarlo (ej: `"number"`, `"string"`, `"object"`).
* **`var`**: Declara una variable, con scope de función (obsoleta, prefiere `let` y `const`).
* **`void`**: Evalúa una expresión y luego devuelve `undefined`. Poco común.
* **`while`**: Crea un bucle que ejecuta un bloque de código mientras una condición sea verdadera.
* **`with`**: Extiende el chain de scope para una sentencia. **Está obsoleta y no debe usarse** por problemas de rendimiento y ambigüedad.

B. Palabras Reservadas para Uso Futuro Estas palabras no tienen功能 actualmente, pero están reservadas para futuras versiones del lenguaje.

* `enum`

C. Palabras Reservadas en Modo Estricto (`"use strict"`) Estas palabras están reservadas solo cuando se usa el modo estricto de ECMAScript 5.

* `implements`
* `interface`
* `let`
* `package`
* `private`
* `protected`
* `public`
* `static`
* `yield`

D. Literales que son Palabras Reservadas Estos son valores, pero también están reservados.

* **`null`**: Representa la ausencia intencional de cualquier valor u objeto.
* **`true` / `false`**: Valores booleanos que representan verdadero y falso.
* **`undefined`**: Indica que una variable no ha sido asignada a un valor.

Parte 2: Métodos Predefinidos en JavaScript

¿Qué es un Método? Un **método** es una función que es propiedad de un objeto. En JavaScript, casi todos los tipos de datos (como Strings, Arrays, Números, etc.) son objetos y, por lo tanto, tienen métodos predefinidos que puedes usar para realizar operaciones sobre ellos.

**Sintaxis:** `objeto.metodo(argumentos)`

Es imposible listar *todos* los métodos porque son cientos, pero aquí están los más importantes y comunes, agrupados por el tipo de objeto al que pertenecen.

1. Métodos de Array Operan sobre arrays (listas).

* **`.push(elemento1, ...)`**: Añade uno o más elementos al **final** de un array y devuelve la nueva longitud.
```javascript
let frutas = ['manzana'];
frutas.push('naranja'); // frutas ahora es ['manzana', 'naranja']
```

* **`.pop()`**: Elimina el **último** elemento de un array y lo devuelve.
```javascript
let ultimaFruta = frutas.pop(); // ultimaFruta es 'naranja', frutas vuelve a ser ['manzana']
```

* **`.shift()`**: Elimina el **primer** elemento de un array y lo devuelve.
```javascript
let primeraFruta = frutas.shift(); // primeraFruta es 'manzana', frutas ahora es []
```

* **`.unshift(elemento1, ...)`**: Añade uno o más elementos al **inicio** de un array y devuelve la nueva longitud.

* **`.map(funcionCallback)`**: Crea un **nuevo array** con los resultados de la llamada a la función proporcionada aplicada a cada uno de sus elementos. Es fundamental para transformar datos.
```javascript
let numeros = [1, 2, 3];
let duplicados = numeros.map(num => num * 2); // duplicados es [2, 4, 6]
```

* **`.filter(funcionCallback)`**: Crea un **nuevo array** con todos los elementos que pasen la prueba (la callback devuelve `true`) implementada por la función proporcionada.
```javascript
let mayoresA1 = numeros.filter(num => num > 1); // mayoresA1 es [2, 3]
```

* **`.find(funcionCallback)`**: Devuelve el **primer elemento** del array que cumpla con la condición de la función de prueba. Muy útil para buscar.

* **`.forEach(funcionCallback)`**: Ejecuta la función proporcionada una vez **por cada elemento** del array. Se usa para iterar, como un `for`.

* **`.includes(elemento)`**: Determina si un array incluye un determinado elemento, devuelve `true` o `false`.

* **`.some(funcionCallback)` / `.every(funcionCallback)`**: Comprueban si al menos uno (`some`) o todos (`every`) los elementos del array cumplen la condición.

* **`.reduce(funcionCallback, valorInicial)`**: Ejecuta una función reductora sobre cada elemento de un array, devolviendo como resultado un **único valor** (puede ser un número, string, objeto, etc.). Muy potente.
```javascript
let suma = numeros.reduce((acumulador, num) => acumulador + num, 0); // suma es 6
```

2. Métodos de String Operan sobre cadenas de texto.

* **`.length`**: (Es una propiedad, no un método) Devuelve la longitud de una cadena.
* **`.toUpperCase()` / `.toLowerCase()`**: Devuelven la cadena convertida a mayúsculas o minúsculas.
* **`.trim()`**: Elimina los espacios en blanco al **inicio y final** de una cadena.
* **`.split(separador)`**: Divide una cadena en un array de subcadenas basándose en un separador especificado.
```javascript
let frase = "Hola mundo";
let palabras = frase.split(" "); // palabras es ['Hola', 'mundo']
```
* **`.includes(cadena)`**: Determina si una cadena contiene otra cadena.
* **`.slice(inicio, fin)`**: Extrae una sección de una cadena y devuelve una nueva cadena.
* **`.replace(cadenaVieja, cadenaNueva)`**: Devuelve una nueva cadena reemplazando un patrón (que puede ser una cadena o una expresión regular) por otra cadena.

3. Métodos de Objeto (`Object`) Métodos estáticos que se llaman directamente sobre el objeto `Object`.

* **`Object.keys(objeto)`**: Devuelve un array con las **propiedades** (keys) enumerables de un objeto.
```javascript
let usuario = { nombre: 'Ana', edad: 30 };
let propiedades = Object.keys(usuario); // propiedades es ['nombre', 'edad']
```
* **`Object.values(objeto)`**: Devuelve un array con los **valores** de las propiedades enumerables de un objeto.
* **`Object.entries(objeto)`**: Devuelve un array de arrays, donde cada subarray es un par `[propiedad, valor]`.
* **`Object.assign(destino, origen)`**: Copia todas las propiedades enumerables de un objeto origen a un objeto destino. Se usa para fusionar objetos.

4. Métodos de Número (`Number`)
* **`Number.isInteger(num)`**: Determina si el valor pasado es un entero.
* **`toFixed(digitos)`**: (Método de instancia) Formatea un número usando notación de punto fijo (para decimales).

5. Métodos para Promesas
* **`.then(funcionExito, funcionFalla)`**: Anexa manejadores de resolución y rechazo a la promesa.
* **`.catch(funcionFalla)`**: Anexa un manejador de rechazo a la promesa. Es la forma preferida de manejar errores.
* **`.finally(funcion)`**: Anexa un manejador que se ejecuta cuando la promesa se resuelve (ya sea con éxito o con fallo). Útil para limpieza.

Conclución Dominar las palabras reservadas te permite entender la estructura del lenguaje, mientras que conocer los métodos predefinidos te da el poder para manipular datos de forma eficiente y elegante. La práctica constante es clave para internalizarlos.

Métodos como Funciones de Objetos

En JavaScript, los métodos son funciones que están asociadas a objetos. Estas funciones pueden acceder y manipular las propiedades del objeto utilizando la palabra clave this, que hace referencia al objeto que contiene el método.

Métodos en objetos literales

// Crear un objeto con métodos
const persona = {
  nombre: "Ana",
  edad: 30,
  // Método que usa propiedades del objeto
  presentarse: function() {
    return `Hola, me llamo ${this.nombre} y tengo ${this.edad} años.`;
  },
  // Método para cambiar una propiedad
  cumplirAños: function() {
    this.edad++;
    return `¡Feliz cumpleaños! Ahora tengo ${this.edad} años.`;
  }
};

console.log(persona.presentarse()); // "Hola, me llamo Ana y tengo 30 años."
console.log(persona.cumplirAños()); // "¡Feliz cumpleaños! Ahora tengo 31 años."
console.log(persona.edad); // 31

Métodos en clases (ES6)

// Definir una clase con métodos
class Rectangulo {
  constructor(ancho, alto) {
    this.ancho = ancho;
    this.alto = alto;
  }
  
  // Método para calcular área
  area() {
    return this.ancho * this.alto;
  }
  
  // Método para calcular perímetro
  perimetro() {
    return 2 * (this.ancho + this.alto);
  }
  
  // Método para cambiar dimensiones
  redimensionar(nuevoAncho, nuevoAlto) {
    this.ancho = nuevoAncho;
    this.alto = nuevoAlto;
    return `Nuevas dimensiones: ${nuevoAncho} x ${nuevoAlto}`;
  }
}

// Crear una instancia y usar sus métodos
const miRectangulo = new Rectangulo(5, 3);
console.log(miRectangulo.area()); // 15
console.log(miRectangulo.perimetro()); // 16
console.log(miRectangulo.redimensionar(7, 4)); // "Nuevas dimensiones: 7 x 4"
console.log(miRectangulo.area()); // 28

Uso de 'this' en métodos

// El contexto de 'this' en métodos
const coche = {
  marca: "Toyota",
  modelo: "Corolla",
  velocidad: 0,
  
  acelerar: function(incremento) {
    this.velocidad += incremento;
    return `Acelerando: ${this.velocidad} km/h`;
  },
  
  frenar: function(decremento) {
    this.velocidad = Math.max(0, this.velocidad - decremento);
    return `Frenando: ${this.velocidad} km/h`;
  },
  
  estado: function() {
    return `${this.marca} ${this.modelo} a ${this.velocidad} km/h`;
  }
};

console.log(coche.estado()); // "Toyota Corolla a 0 km/h"
console.log(coche.acelerar(50)); // "Acelerando: 50 km/h"
console.log(coche.acelerar(30)); // "Acelerando: 80 km/h"
console.log(coche.frenar(40)); // "Frenando: 40 km/h"
console.log(coche.estado()); // "Toyota Corolla a 40 km/h"

// Problemas comunes con 'this'
const obtenerEstado = coche.estado;
// console.log(obtenerEstado()); // Error: 'this' es undefined

// Solución: usar bind para fijar el contexto
const estadoBind = coche.estado.bind(coche);
console.log(estadoBind()); // "Toyota Corolla a 40 km/h"

💡 Buenas prácticas con métodos de objetos

  • Usa métodos para agrupar funcionalidades relacionadas con un objeto
  • Considera usar clases para objetos complejos con múltiples métodos
  • Sé consciente del contexto de this - puede cambiar según cómo se llame al método
  • Usa bind(), call() o apply() cuando necesites controlar el contexto de this
  • Para métodos simples, considera la sintaxis abreviada de métodos en ES6

const calculadora = {
  resultado: 0,
  
  sumar(valor) {
    this.resultado += valor;
    return this;
  },
  
  restar(valor) {
    this.resultado -= valor;
    return this;
  },
  
  multiplicar(valor) {
    this.resultado *= valor;
    return this;
  },
  
  dividir(valor) {
    this.resultado /= valor;
    return this;
  },
  
  valor() {
    return this.resultado;

Métodos estáticos en clases

class Matemáticas {
  // Método estático - pertenece a la clase, no a las instancias
  static esPar(numero) {
    return numero % 2 === 0;
  }
  
  static esPrimo(numero) {
    if (numero <= 1) return false;
    if (numero <= 3) return true;
    
    if (numero % 2 === 0 || numero % 3 === 0) return false;
    
    for (let i = 5; i * i <= numero; i += 6) {
      if (numero % i === 0 || numero % (i + 2) === 0) return false;
    }
    
    return true;
  }
  
  static maximo(...numeros) {
    return Math.max(...numeros);
  }
  
  static minimo(...numeros) {
    return Math.min(...numeros);
  }
}

// Los métodos estáticos se llaman directamente en la clase
console.log(Matemáticas.esPar(10)); // true
console.log(Matemáticas.esPrimo(17)); // true
console.log(Matemáticas.maximo(3, 8, 2, 10)); // 10
console.log(Matemáticas.minimo(3, 8, 2, 10)); // 2

// No están disponibles en las instancias
const math = new Matemáticas();

Manipulación de Arrays en JavaScript

Los arrays son estructuras de datos fundamentales en JavaScript que permiten almacenar colecciones de elementos. La manipulación de arrays se refiere a todas las operaciones que podemos realizar sobre ellos: agregar, eliminar, buscar, transformar y recorrer elementos. JavaScript ofrece métodos poderosos para trabajar con arrays de forma eficiente y expresiva.

💡 ¿Por qué son importantes los arrays?

Los arrays son esenciales porque:

  • Almacenan múltiples valores en una sola variable
  • Permiten organizar datos de manera estructurada
  • Ofrecen métodos poderosos para manipular datos
  • Son la base para trabajar con listas, tablas y conjuntos de datos
  • Permiten algoritmos eficientes de búsqueda y ordenamiento

Creación de Arrays

// Diferentes formas de crear arrays
let frutas = ["Manzana", "Banana", "Naranja"]; // Literal (recomendado)
let numeros = new Array(1, 2, 3, 4, 5); // Constructor
let vacio = []; // Array vacío
let mixto = [1, "Hola", true, {nombre: "Juan"}]; // Tipos mixtos

console.log(frutas); // ["Manzana", "Banana", "Naranja"]
console.log(numeros); // [1, 2, 3, 4, 5]
console.log(vacio); // []
console.log(mixto); // [1, "Hola", true, {nombre: "Juan"}]

Acceso y modificación de elementos

let colores = ["Rojo", "Verde", "Azul"];

// Acceder a elementos por índice (0-based)
console.log(colores[0]); // "Rojo"
console.log(colores[2]); // "Azul"

// Modificar elementos
colores[1] = "Amarillo";
console.log(colores); // ["Rojo", "Amarillo", "Azul"]

// Longitud del array
console.log(colores.length); // 3

// Acceder al último elemento
console.log(colores[colores.length - 1]); // "Azul"

// Índices fuera de rango
console.log(colores[5]); // undefined

Métodos principales para manipular arrays

Agregar y eliminar elementos

let animales = ["Perro", "Gato"];

// Agregar al final
animales.push("Conejo");
console.log(animales); // ["Perro", "Gato", "Conejo"]

// Eliminar del final
let eliminado = animales.pop();
console.log(eliminado); // "Conejo"
console.log(animales); // ["Perro", "Gato"]

// Agregar al inicio
animales.unshift("Pájaro");
console.log(animales); // ["Pájaro", "Perro", "Gato"]

// Eliminar del inicio
eliminado = animales.shift();
console.log(eliminado); // "Pájaro"
console.log(animales); // ["Perro", "Gato"]

// Eliminar/agregar en cualquier posición
animales.splice(1, 0, "Hamster", "Pez"); // Agrega en posición 1
console.log(animales); // ["Perro", "Hamster", "Pez", "Gato"]

animales.splice(1, 2); // Elimina 2 elementos desde posición 1
console.log(animales); // ["Perro", "Gato"]

Buscar elementos

let numeros = [10, 20, 30, 40, 50, 30];

// indexOf - encuentra primer índice
console.log(numeros.indexOf(30)); // 2
console.log(numeros.indexOf(100)); // -1 (no encontrado)

// lastIndexOf - encuentra último índice
console.log(numeros.lastIndexOf(30)); // 5

// includes - verifica existencia
console.log(numeros.includes(20)); // true
console.log(numeros.includes(99)); // false

// find - encuentra primer elemento que cumpla condición
let mayor25 = numeros.find(num => num > 25);
console.log(mayor25); // 30

// findIndex - encuentra índice del primer elemento que cumpla condición
let indice = numeros.findIndex(num => num > 35);

POO y Clases en JavaScript

La Programación Orientada a Objetos (POO) es un paradigma que organiza el código en "objetos" que contienen tanto datos (propiedades) como comportamientos (métodos). JavaScript implementa POO a través de prototipos, pero desde ES6 también incluye sintaxis de clases que facilita su uso.

Las clases son plantillas para crear objetos, encapsulando datos y funcionalidades. El keyword this se refiere al contexto actual de ejecución, generalmente la instancia del objeto cuando se usa dentro de métodos de clase.

class CuentaBancaria {

        this.titular = titular;
        this.#saldo = saldoInicial;
        this.#pin = pin;

    #validarPin(pin) {
        return this.#pin === pin;
    }
    
    depositar(cantidad) {
        if (cantidad > 0) {
            this.#saldo += cantidad;
            return `Depositado: $${cantidad}. Nuevo saldo: $${this.#saldo}`;
        }
        return "Cantidad no válida";
    }
    
    retirar(cantidad, pin) {

            return "PIN incorrecto";
        }
        
        if (cantidad > 0 && cantidad <= this.#saldo) {
            this.#saldo -= cantidad;
            return `Retirado: $${cantidad}. Nuevo saldo: $${this.#saldo}`;
        }
        return "Fondos insuficientes o cantidad no válida";
    }
}

class EjemploThis {
    constructor(valor) {
        this.valor = valor;
        this.metodoNormal = this.metodoNormal.bind(this);
    }
    
    metodoNormal() {
        console.log("this en metodoNormal:", this.valor);
    }
    
    metodoFlecha = () => {
        // Las arrow functions heredan el 'this' del contexto donde se definieron
        console.log("this en metodoFlecha:", this.valor);
    }
    
    probarThis() {
        console.log("--- Contexto original ---");
        this.metodoNormal(); // this = instancia
        this.metodoFlecha(); // this = instancia
        
        console.log("--- Contexto perdido ---");
        const fnNormal = this.metodoNormal;
        const fnFlecha = this.metodoFlecha;
        
        try {
            fnNormal(); // Error: this es undefined (en modo estricto)
        } catch (e) {
            console.log("Error con metodoNormal:", e.message);
        }
        
        fnFlecha(); // Funciona: this = instancia (arrow function)
        
        console.log("--- Con bind ---");
        const fnNormalBind = this.metodoNormal.bind(this);
        fnNormalBind(); // Funciona: this = instancia (gracias al bind en constructor)
    }
}

Manejo de Errores

El manejo de errores es una parte fundamental de la programación en JavaScript. Permite controlar situaciones excepcionales y evitar que nuestra aplicación se detenga abruptamente. JavaScript proporciona el bloque try...catch para manejar errores de manera elegante, así como el objeto Error para crear y lanzar errores personalizados.

Bloque try...catch básico

try {
    // Código que puede generar un error
    let resultado = unaFuncionQueNoExiste();
    console.log(resultado);
} catch (error) {
    // Manejo del error
    console.error("Ocurrió un error:", error.message);
} finally {
    // Este bloque siempre se ejecuta
    console.log("Finalizando proceso...");
}

Error personalizado con throw

function dividir(a, b) {
    if (b === 0) {
        throw new Error("No se puede dividir por cero");
    }
    return a / b;
}

try {
    let resultado = dividir(10, 0);
    console.log("Resultado:", resultado);
} catch (error) {
    console.error("Error al dividir:", error.message);
}

Tipos de errores específicos

try {
    // Diferentes tipos de errores
    null.nombre; // TypeError
} catch (error) {
    if (error instanceof TypeError) {
        console.error("Error de tipo:", error.message);
    } else if (error instanceof ReferenceError) {
        console.error("Error de referencia:", error.message);
    } else {
        console.error("Error desconocido:", error.message);
    }
}

💡 Buenas prácticas en manejo de errores

- Siempre maneja errores específicos en lugar de usar catch genéricos.
- Proporciona mensajes de error claros y útiles para el debugging.
- Usa finally para limpieza de recursos sin importar el resultado.
- Considera usar console.error() en lugar de console.log() para errores.
- No abuses del manejo de errores para controlar el flujo normal de tu programa.

Los módulos ES6 son una forma nativa de organizar código JavaScript en archivos separados. Permiten exportar e importar funcionalidades entre diferentes archivos, facilitando la estructuración de aplicaciones complejas y la reutilización de código.
